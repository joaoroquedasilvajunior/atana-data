# Methodology — `canonical.domain_crosswalk`

The Atana harmonisation crosswalk. Phase 3 (Part C) of the Atana Data LATAM
expansion — the table that turns four isolated national schemas into a genuinely
cross-queryable layer.

**Schema / table:** `atana.canonical.domain_crosswalk`
**File:** `curated/domain_crosswalk.parquet` (+ `.meta.json`)
**Build script:** `etl/canonical__build_domain_crosswalk.py`
**Built:** 2026-05-22 · **extended:** 2026-05-23 (Phase 4 — the `ibge_siic` rows).

---

## 1. What this is

A single reference table that maps **every cultural-statistics classification in
the corpus** onto one common spine.

The corpus holds eight classification systems that do not talk to each other:

- **INEGI CSCM** (Mexico) — 10 *áreas generales*
- **DANE CSECC** (Colombia) — 35 product *cuadros* (22 carry trade)
- **SInCA CSC** (Argentina) — 2 segments (goods / services)
- **CSCCR** (Costa Rica) — 4 sectors
- **UNCTAD CER** — 8 creative-goods codes + 7 creative-services codes
- **IBGE NCM** (Brazil) — chapter-level (the 5 pure-cultural chapters)
- **IBGE SIIC** (Brazil) — the *domínios culturais* classification, added in
  Phase 4 (chapters 1 / 2 / 7 / 9)
- **BCB SGS** (Brazil) — the balance-of-payments IP-services account, added in
  Phase 4c

The crosswalk maps all of them onto the **14 domains of the 2025 UNESCO
Framework for Cultural Statistics (FCS)** — 7 cultural + 7 transversal — which
is the spine Atana adopted as its methodological anchor (Atana Note #05).

It exists so a cross-country query can resolve through **one table** instead of
six bespoke joins — *while keeping the definitional gaps visible*. It is the
operational form of the methodological-pluralism principle: where two systems
disagree, the disagreement is recorded (in `mapping_confidence` and `notes`),
never silently averaged away (cf. Atana Note #03, the "55.6% vs 5.0%" trap).

It is **not raw source data.** It is a curated artifact — the inline data in the
build script is transcribed from the Phase 3 and Phase 4 methodology docs. It
lives in the `canonical` schema and the `curated/` folder, and it is
**un-timestamped**:
a living reference table regenerated by its build script, not a versioned
analytical snapshot.

## 2. The table

One row per classification code. **83 rows** — Phase 3 built 72; Phase 4 added
10 `ibge_siic` rows and 1 `bcb` row.

| Column | Type | Description |
|---|---|---|
| `source_schema` | VARCHAR | `fcs2025` / `inegi` / `dane` / `sinca` / `cr_bccr` / `unctad` / `ibge_comex` / `ibge_siic` / `bcb` |
| `source_system` | VARCHAR | Human-readable classification name (e.g. `CSCM área general`) |
| `source_code` | VARCHAR | The code within that classification (`5`, `CER010`, `49`) |
| `source_label` | VARCHAR | The label within that classification |
| `fcs2025_domain` | VARCHAR | The common spine — an FCS 2025 domain, a **bundle**, or NULL |
| `fcs2025_domain_type` | VARCHAR | `cultural` / `transversal` (or a verbatim bundle / NULL) |
| `unctad_cer` | VARCHAR | Nearest UNCTAD CER / service code; NULL if no equivalent |
| `ibge_ncm_chapter` | VARCHAR | Nearest IBGE NCM chapter(s); NULL if not a traded good |
| `mapping_confidence` | VARCHAR | `exact` / `good` / `approximate` / `no-equivalent` |
| `notes` | VARCHAR | The definitional gap, stated explicitly. `★` flags a finding |

**`mapping_confidence` scale.** `exact` — the FCS spine rows themselves (a
domain maps to itself). `good` — a clean, near-1:1 correspondence. `approximate`
— a real but lossy map (bundling, partial overlap, a wider/narrower national
definition). `no-equivalent` — the classification code has no FCS cultural
domain at all; these rows carry a NULL `fcs2025_domain`.

**Bundle values.** Where a national or UNCTAD code genuinely spans several FCS
domains, `fcs2025_domain` holds a bundle string rather than a false 1:1: a `;`-
or `/`-joined list (e.g. `Music; Performing arts; Visual arts and crafts` for
UNCTAD CER040) or `Multiple — not separable` / `Multiple — all creative
services` (Argentina's segment-level data, the UNCTAD service aggregate). A
query that wants clean single-domain rows should filter these out explicitly —
the bundle is the honest representation of a many-to-many.

## 3. The 2025 FCS spine

The 14 spine rows (`source_schema = 'fcs2025'`) are the harmonisation target.

**7 cultural domains:** Cultural and natural heritage · Performing arts · Visual
arts and crafts · Books and press · Audiovisual · Design · Music.

**7 transversal domains:** Education in culture · Archives · Cultural and
creative goods manufacturing · ICT and digital infrastructure · Intangible
cultural heritage · Social participation · Intellectual property.

The transversal domains are a 2025-FCS addition (the praxeological lens) — they
capture cultural activity that cuts across the cultural domains rather than
sitting inside one.

## 4. Row composition (83 rows)

| `source_schema` | Rows | What |
|---|--:|---|
| `fcs2025` | 14 | The FCS spine itself — 7 cultural + 7 transversal domains |
| `inegi` | 10 | CSCM áreas generales (`inegi_csc.md` §5) |
| `dane` | 22 | CSECC trade-bearing product cuadros (`dane_csecc.md` §4) |
| `sinca` | 2 | SInCA CSC segments — goods, services (`sinca_csc.md` §5) |
| `cr_bccr` | 4 | CSCCR sectors (`cr_bccr_csc.md` §5) |
| `unctad` | 15 | 8 UNCTAD CER creative-goods codes + 7 creative-services codes |
| `ibge_comex` | 5 | The 5 pure (100%) cultural NCM chapters — 37, 46, 49, 92, 97 |
| `ibge_siic` | 10 | IBGE SIIC *domínios culturais* — Phase 4 (chapters 1/2/7/9) |
| `bcb` | 1 | BCB SGS balance-of-payments IP-services account — Phase 4c.1 |

The IBGE rows are deliberately limited to the **five pure-cultural NCM
chapters** (the `is_pure_cultural` flag in `ibge_comex.tab_10_1`). The
partial-cultural chapters (32, 39, 44, 67, 69, 71, 83–85, 90, 91, 94, 95) are
not enumerated as crosswalk rows — they are partial by construction and a
chapter-level crosswalk row would imply a cleaner map than exists.

## 5. How to use it

The point of the table is the join. To find every classification that touches
a given FCS domain:

```sql
SELECT source_schema, source_code, source_label, mapping_confidence
FROM   atana.canonical.domain_crosswalk
WHERE  fcs2025_domain = 'Audiovisual'
ORDER  BY source_schema, source_code;
```

To resolve a national figure to the spine — e.g. tag Mexico's CSCM trade rows
with their FCS domain:

```sql
SELECT c.*, x.fcs2025_domain, x.fcs2025_domain_type, x.mapping_confidence
FROM   atana.inegi.csc_comercio          c
JOIN   atana.canonical.domain_crosswalk  x
  ON   x.source_schema = 'inegi'
 AND   x.source_code   = CAST(c.area_code AS VARCHAR);
```

**Always carry `mapping_confidence` through** to the result. A cross-country
comparison built only on `good`/`exact` rows is defensible; one that silently
includes `approximate` bundles is not — surface the confidence so the reader
sees it.

## 6. The definitional gaps — the findings

Eleven rows carry a `★` in `notes`. They are not noise; they are publishable
findings, the way Atana Note #03 treated UNCTAD vs IBGE Comex:

- **Mexican artesanías.** Crafts are 18.4% of Mexico's cultural GDP yet mostly
  informal household production. UNCTAD CER020 sees only the traded-goods
  fraction; in `inegi.csc_comercio` the área records `NA`. Mexico's
  cultural-trade total structurally excludes its largest domain.
- **Digital content straddle.** INEGI's *Contenidos digitales e internet* — its
  fastest-growing área — sits across the cultural Audiovisual domain (streaming)
  and the transversal ICT domain (infrastructure). Where the boundary falls
  moves Mexico's "cultural GDP" by a percentage point.
- **Advertising / architecture.** A three-way many-to-many: INEGI and DANE
  bundle advertising and architecture into design areas; UNCTAD routes
  advertising through the SAMA *service* code and architecture through CER050;
  FCS treats Design as one domain. Four rows (`inegi` #7, `dane` 33 & 35,
  `cr_bccr` Publicidad, plus `unctad` CER040's three-arts bundle) carry this.
- **Brazil's participation proxy.** Phase 4 reaches *Social participation* only
  through digital-access and leisure-travel proxies (`ibge_siic` rows
  `CH7-TIC`, `CH9-TUR`, both `approximate`). Brazil has no continuous national
  cultural-practices survey — unlike Colombia's *Encuesta de Consumo Cultural*
  or Argentina's *Encuesta Nacional de Consumos Culturales*. The
  under-measurement is itself the finding.

**The coverage meter — 13 of 14, after Phase 4.** Phase 3 left a structural
gap: of the 14 FCS domains, only **10 were reached by a national CSC trade
row** — the four missing ones (*Cultural and creative goods manufacturing*,
*Intangible cultural heritage*, *Intellectual property*, *Social
participation*) being exactly the transversal domains a *foreign-trade* module
cannot see. **Phase 4 (Brazil-first) closes three of them.** The 10 `ibge_siic`
rows add the IBGE SIIC cultural-domain classification: SIIC chapter 2's
structural surveys reach *Cultural and creative goods manufacturing* (the
`PERIF-MANUF` row), and the PNADC ICT and leisure-tourism supplements reach
*Social participation* (`CH7-TIC`, `CH9-TUR`) — the latter as an explicit
`approximate` **proxy**. The `bcb` row (Phase 4c.1) adds the BCB
balance-of-payments IP-services account, reaching *Intellectual property*. The
build script's coverage meter now prints **13/14**. Only *Intangible cultural
heritage* — registry and inventory data only, out of scope by decision —
remains unreached.

## 7. Limitations and caveats

- **Trade-module scope.** The national rows describe the *classifications used
  in the trade modules* ingested in Phase 3. They are not a full crosswalk of
  each country's entire CSC.
- **Argentina is coarse by necessity.** SInCA's trade data is segment-level
  only — two rows, both `Multiple — not separable`. A fine Argentine crosswalk
  is impossible at the trade-module grain (see `sinca_csc.md` §5).
- **`approximate` dominates** — 44 of 83 rows. That is the honest state of
  cross-national cultural statistics, not a defect of this table. The source
  systems use distinct classifications, valuation conventions and currencies;
  an `approximate` map is often the *best* map that exists.
- **The crosswalk does not reconcile currencies, valuations or price bases.**
  It harmonises *domains* only. Mixing the underlying figures still requires the
  per-source reconciliation each methodology doc spells out.
- **UNCTAD CER codes** are transcribed from the corpus' `unctad` schema and the
  Phase 3 design doc; re-verify leaf-code detail (CER021–CER027) against the
  live UNCTAD taxonomy before any publication leans on it.

## 8. Regeneration

```bash
python etl/canonical__build_domain_crosswalk.py
```

The build is fully inline-data-driven and idempotent — reruns produce a
byte-identical Parquet. To revise a mapping, edit the relevant block in the
build script (`INEGI`, `DANE`, `SINCA`, `CR_BCCR`, `UNCTAD_GOODS`,
`UNCTAD_SERVICES`, `IBGE_NCM`, `SIIC`, `BCB`, or the `FCS_*` spine) and re-run;
the script
self-validates row counts, the confidence vocabulary, spine membership and the
NULL/`no-equivalent` invariant before writing.
